Regex-based rules use Python regular expressions. Prefix with (?i) to ignore case. For JSON path rules, use dot-separated keys such as metadata.merchant.type – the rule fires when that path exists and has a truthy value.

Date ranges are optional. When provided, the rule applies only to transactions whose timestamp falls within the inclusive window.
